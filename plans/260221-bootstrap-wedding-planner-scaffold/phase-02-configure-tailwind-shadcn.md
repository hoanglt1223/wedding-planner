## Overview

- **Priority:** P1 (blocking — required for UI)
- **Status:** Pending
- **Description:** Install Tailwind CSS v4 with the `@tailwindcss/vite` plugin (CSS-first, no PostCSS), configure path aliases, initialize shadcn/ui, and install starter components.

## Key Insights

- Tailwind v4 uses `@tailwindcss/vite` plugin — no `postcss.config.js` or `tailwind.config.js` needed
- CSS entry: single `@import "tailwindcss"` replaces old `@tailwind` directives
- Path aliases (`@/*`) must be configured in BOTH `tsconfig.json` AND `tsconfig.app.json`
- `vite.config.ts` needs `resolve.alias` for `@` to resolve at build time
- shadcn/ui init generates `components.json` and `src/lib/utils.ts` (cn utility)
- Radix UI now ships as unified `radix-ui` package (Feb 2026)

## Requirements

### Functional
- Tailwind v4 utilities work in JSX (`className="bg-red-500"` renders correctly)
- shadcn/ui components render with correct styling
- Path alias `@/` resolves to `./src/` in imports
- Starter components installed: button, card, input, label, separator, badge

### Non-Functional
- No PostCSS config file (CSS-first approach)
- No `tailwind.config.js` file
- Build completes without CSS warnings

## Architecture

```
vite.config.ts
  └── plugins: [react(), tailwindcss()]
  └── resolve.alias: { "@": "./src" }

src/index.css
  └── @import "tailwindcss"
  └── (shadcn CSS variables injected by init)

tsconfig.json + tsconfig.app.json
  └── compilerOptions.baseUrl: "."
  └── compilerOptions.paths: { "@/*": ["./src/*"] }

components.json (generated by shadcn init)
  └── style: "new-york"
  └── baseColor: "neutral"
  └── aliases: { components: "@/components", lib: "@/lib" }
```

## Implementation Steps

1. **Install Tailwind CSS v4 + Vite plugin**
   ```bash
   npm install tailwindcss @tailwindcss/vite
   ```

2. **Update `vite.config.ts`**
   ```typescript
   import path from "path"
   import tailwindcss from "@tailwindcss/vite"
   import react from "@vitejs/plugin-react"
   import { defineConfig } from "vite"

   export default defineConfig({
     plugins: [react(), tailwindcss()],
     resolve: {
       alias: {
         "@": path.resolve(__dirname, "./src"),
       },
     },
   })
   ```

3. **Update `tsconfig.json`** — add to `compilerOptions`:
   ```json
   {
     "compilerOptions": {
       "baseUrl": ".",
       "paths": {
         "@/*": ["./src/*"]
       }
     }
   }
   ```

4. **Update `tsconfig.app.json`** — add same `baseUrl` and `paths` to `compilerOptions`

5. **Replace `src/index.css` contents**
   ```css
   @import "tailwindcss";
   ```
   - Remove all existing Vite default styles
   - shadcn init will append CSS variables below this import

6. **Initialize shadcn/ui**
   ```bash
   npx shadcn@latest init
   ```
   - When prompted:
     - Style: **New York**
     - Base color: **Neutral**
     - CSS variables for theming: **Yes**
   - This generates `components.json` and `src/lib/utils.ts`

7. **Install starter components**
   ```bash
   npx shadcn@latest add button card input label separator badge
   ```

8. **Verify build**
   ```bash
   npm run build
   ```
   - Should complete with zero errors
   - Check that `dist/` output includes CSS with Tailwind utilities

9. **Verify dev server renders Tailwind styles**
   ```bash
   npm run dev
   ```
   - Temporarily add a Tailwind class to `App.tsx` (e.g., `<h1 className="text-3xl font-bold">`)
   - Confirm styling applies
